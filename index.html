Question:

How does javaScript work? What does javaScript event loop do? Local storage and session storage difference?

JavaScript is a type of programming language. It basically works by v8 engine. We are pouring our JavaScript code as input and the v8 engine is processing our code well and showing our output in the browser.v8 The engine performs these 4 steps after receiving any code input 1. parsing 2. compiling 3. executing and 4, optimizing shows our output in browser.
JavaScript has two types of events: 1. synchronous events 2. asynchronous events. Event loops usually deal with asynchronous events. Whenever an async function or event is called, it is sent to a browser API. Based on the command received from the call stack, the API starts its own single-threaded operation. An example of this is the setTimeout method.
Session storage is temporary which exists as long as the current window is opened and data gets lost when window is closed where as Local Storage is persistant and remains intact across sessions unless anyone deletes it. A new session is created each time a window or tab is opened. Closing a window or tab ends the session storage.
